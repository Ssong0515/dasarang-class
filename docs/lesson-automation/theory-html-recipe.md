# 이론 슬라이드 HTML 생성 레시피 (Claude 직접 생성용)

> **입력**: 회차 `details` 1건(시수별). **출력**: 시수마다 자체 완결 HTML 이론 슬라이드 덱 1개.
> 이 문서가 곧 **생성 지침**이다. 실습 레시피 [`practice-recipe.md`](./practice-recipe.md)와 **짝**이며, 디자인 언어를 공유해 이론↔실습 톤을 맞춘다.
>
> **왜 HTML 이론 덱?** 정식 이론 경로([`notebooklm-guidelines.md`](./notebooklm-guidelines.md))는 **NotebookLM 수동 제작**이다(공개 API 없음). 그래서 자동화가 안 되고, 강사가 매번 손으로 만들어야 한다. 이 레시피는 **Claude가 이론을 자체 완결 HTML 덱으로 직접 생성**하게 해서, 향후 **이론+실습을 한 루틴에서 함께 자동 생성**하는 것을 목표로 한다(=실습은 `practice-recipe.md`, 이론은 이 문서).
>
> **트레이드오프(정직하게)**: HTML 덱은 *발표 즉시 가능·자동 생성·렌더링 검증 가능*이 강점. 단 **이모지 + CSS/SVG로만** 그려야 한다(자체 완결·외부 리소스 0). **진짜 윈도우 스크린샷·AI 일러스트가 필요하면 NotebookLM 경로가 낫다**. 둘은 대체가 아니라 선택지.
>
> **품질 기준선**: [`sample-theory-files-and-storage.html`](./sample-theory-files-and-storage.html) (2회차 심화 "파일의 실제 위치와 저장 구조", 16장, 시각 우선 + 러시아어 병기). **코드 패턴(스테이지 셸·네비·창 목업)·디자인 토큰은 이 샘플을 그대로 참고**한다.

## 0. 대상 학습자 (실습과 동일)

- **9~24세 외국인(이주민) 학생, 한국어 배우는 중, 컴퓨터 처음.** 난이도 중학생 수준. ❌어르신 아님.
- → **🎯 시각 우선 / 한국어 텍스트 최소화(가장 중요)**, 큰 글씨, 쉬운 짧은 한국어, **핵심 문구에 모국어 병기**, 따뜻한 톤.

## 1. 내용 출처 = 커리큘럼 `details` (★ 실습HTML·옛 md 금지)

- `list_resource(curriculums)` → 해당 회차 `details` 파싱. **`curriculum-by-class.md`(옛 3~5월 커리큘럼)나 실습 HTML 결과물을 출처로 쓰지 말 것.**
- 현재 컴퓨터 수업 커리큘럼: **`3u4YLYCstBfvW4sz5occ` "디지털 첫걸음에서 AI 활용까지"(20회차)** — 디지털.AI 문해교육 1·2반.
- **1회차 = 2시수 = 앞 40분 기초 + 뒤 40분 심화.** 시수당 1이론 + 1실습. `details`에서 시수별 **(시수제목/모듈/가르칠 내용/핵심 개념/정리 문장)** 추출. 이론↔실습 시수 매핑은 `practice-recipe.md` 부록(40시수 매핑표) 참고.
- 예) "내 이름 학습 폴더 만들기" 실습 = 2회차 심화 → 그 이론 = "파일의 실제 위치와 저장 구조".

## 2. 16장 구조 (notebooklm-guidelines §4 정본 그대로)

표지 → 오늘의 목표 → 왜 필요할까 → 오늘의 단어 → 개념① 소개·자세히 → 개념② 소개·자세히 → 개념③ 소개 → 한눈에 정리 → 화면 둘러보기 → 순서 한눈에 → 조심할 점 → **결과물 미리보기** → 이해 확인(OX/고르기) → **실습으로 연결**.

- ⚠️ **순수 이론(설명)까지만.** 학생이 직접 따라 하는 실습 단계·완료 확인 기준은 넣지 않는다(그건 실습 HTML 담당, 16장에서 연결만).
- ⚠️ **마무리 인사 슬라이드 없음**(앱 "수업 마치기" 버튼이 대체).
- 개념이 3개보다 많으면 화면 둘러보기·순서·조심할 점 장에 녹인다(샘플은 6개 가르칠 내용을 개념①②③ + 화면/순서/주의로 분배).

## 3. 시각 우선 (가장 중요)

- 설명 문장 대신 **큰 아이콘 + 도식 + 짧은 라벨(2~4단어)**. 한 장 = 핵심 1개.
- **화면은 CSS로 "진짜 창"처럼**: 제목표시줄(📁 폴더명 + 빨강 ✕) + 주소표시줄(경로 브레드크럼) + 파일 아이콘 격자. (샘플 `.win`/`.win-top`/`.win-addr`/`.win-body`)
- 도식 패턴: **≠ / VS / →흐름 / ✅❌ / before-after / 브레드크럼(내 PC › 문서 › …)**.
- 버튼·메뉴 이름은 `[대괄호]` 박스. 추상 개념(경로·바로가기 등)은 짧은 라벨 + 병기로 보완.

## 4. 모국어 병기 (슬라이드에 직접 인쇄)

- 슬라이드는 **교차출처 임베드**라 앱 🌐 번역 버튼이 안 먹는다 → **번역을 슬라이드에 직접 박는다**(런타임 사전 X). 한국어 크게 + **작은 회색 모국어**를 바로 아래/옆에. (샘플 `.ru` 클래스)
- **언어 = `get_overview`의 반별 `topLanguages` 최대 2개.** 비어 있으면 병기 생략(쉬운 한국어 + 그림만). DB가 비어 있어도 **강사 지정값이 있으면 그걸 쓴다.**
  - **현재: 디지털.AI 1·2반 = 러시아어(강사 지정, 2026-06-29).** DB `topLanguages`는 아직 빈 값 → 채워지면 그걸 우선.
- **병기 대상**: 제목·핵심 라벨·용어·정답 등 핵심 문구만(한 장이 글자로 가득 차지 않게). **병기 제외**: 확장자(.jpg)·영문 용어·실제 버튼 이름([대괄호])·학생이 입력할 한글 단어 — 한글/원문 그대로.

## 5. 디자인 토큰 (notebooklm-guidelines §1 + 실습과 공유)

- **밝은 배경**(흰색/아주 연한 회색). **강조 1색 = 파랑 `#2f6bd8`**. 의미색: 초록 `#1f8a4c`(완료·연결), 주황 `#c4691b`(주의), 빨강 `#c4382f`(금지/오답). 글자 `#23272f`, 보조 `#6b7280`, 병기 회색 `#9aa3b2`.
- **큰 글씨**(제목 ~44px, 본문 라벨 ~24~30px, 병기 ~0.5em), **넉넉한 여백**, 둥근 카드(반경 ~20px), 부드러운 그림자.
- **애니메이션 최소**(슬라이드 전환은 페이드만, `prefers-reduced-motion` 존중). **진행 표시 n/16** 항상.
- 폰트: `"Malgun Gothic","맑은 고딕","Apple SD Gothic Neo",system-ui` (CJK 웹폰트 인라인 비현실적 → 시스템 폰트 의도적 사용. 교실 Windows·검증 Mac 양쪽 OK).
- ⚠️ 피할 것: 다크 히어로 중심, 복잡한 그라데이션, 과한 장식, 초등학생용처럼 유치한 디자인, 제목 밑줄/장식 바.

## 6. 덱 셸 (재사용 — 샘플 코드 그대로)

- **16:9 스테이지 1280×720**를 `transform: scale(min(vw/1280, vh/720))`로 화면맞춤(전체화면 발표 최적, 넘침 0).
- 네비: **←→·Space·PageUp/Down·Home/End·F(전체화면)**, 좌/우 클릭, 스와이프, 하단 점. **컨트롤 바 자동 숨김**(2.8s, 마우스 움직이면 표시) — 발표 중 콘텐츠와 안 겹침.
- 세로 정렬: 콘텐츠 블록은 `margin:auto 0`(또는 flex `min-height:0`)로 **수직 중앙**. (텍스트 줄 수가 바뀌어도 균형 유지)
- **자체 완결**: 외부 리소스 0(폰트·이미지·스크립트 CDN 금지). 이미지는 **이모지 + CSS/SVG만**(Artifact CSP 안전). ~900KB 이내.

## 7. 출력·배포

- **시수마다 HTML 1개.** `<title>` + `<style>` + 마크업 + `<script>` 자체 완결.
- 발행: **claude.ai/code Artifact**(즉시 발표·공유 가능) 또는 파일. ⚠️ Artifact는 외부/생성 이미지 불가 — 같은 캡처-동기화 한계로 QA 시 리로드 후엔 서버 재시작 권장.
- **(향후 루틴) 이론 HTML 저장 경로는 TBD.** 실습은 `create_practice_content`로 학생 콘텐츠(게이팅)에 들어가지만, 이론은 학생용 인터랙티브가 아니라 **강사 발표용**이라 저장 위치·게이팅을 따로 정해야 한다(강사 대시보드 '이론 수업' 칸? 별도 컬렉션?). 현재는 미정.

## 8. 향후 방향 — Claude로 이론+실습 함께 생성하는 루틴 (계획)

- 목표: 새벽 클라우드 루틴이 다가오는 회차를 골라(`practice-recipe.md` §B 선택 로직), `details` 파싱 → **시수마다 이론 HTML(이 레시피) + 실습 HTML(`practice-recipe.md`)을 한 번에 생성**.
- 이점: 이론·실습이 같은 시수 `details` 한 출처에서 나와 **내용·디자인이 자동 일치**. NotebookLM 수동 단계 제거.
- 정하기: 이론 HTML 저장/게이팅 경로(§7), 이론에도 병기 언어 자동 주입(반 `topLanguages`), 실패/재시도.
- 현재 단계: **샘플로 품질 검증 중**(이 문서 + `sample-theory-files-and-storage.html`). 루틴화 전 추가 테스트 예정.
