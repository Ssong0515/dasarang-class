# NotebookLM 슬라이드 공통 지침 (이론 슬라이드)

> **대안 경로(2026-06-29):** 이론을 **Claude가 자체 완결 HTML 덱으로 직접 생성**하는 자동화 레시피가 추가됨 → [`theory-html-recipe.md`](./theory-html-recipe.md) (디자인 토큰·16장 구조는 이 문서 §1·§4를 공유, 시각 우선 + 모국어 병기). 풍부한 일러스트·실제 스크린샷이 필요하면 아래 NotebookLM 수동 경로가 낫다.

> 이론 슬라이드는 **NotebookLM으로 수동 제작**한다(공개 API 없음 → 자동화 제외).
> 운영: **[1] 디자인 + [2] 형식·언어** 지침을 NotebookLM 소스(또는 커스터마이즈 칸)에 한 번 넣고, 매번 **[3] 생성 프롬프트** + **[4] 시수별 내용 프롬프트**를 입력 칸에 붙인다.
> 슬라이드는 교사 발표용이자 **학생이 보고 바로 따라 하는** 자료이며, 이어지는 **실습(인터랙티브 HTML, [`practice-recipe.md`](./practice-recipe.md))** 으로 연결된다. 대상·톤은 실습과 동일.
>
> **출처/정정**: 사용자의 「NotebookLM 슬라이드 디자인 지침」 + 「슬라이드 생성 프롬프트」 문서를 정리한 것. **(1) 병기 번역 부활(2026-06-25)**: 학생은 슬라이드에 크롬 번역을 쓸 수 없다(슬라이드는 구글 슬라이드 등 **교차출처 임베드**라 크롬 페이지 번역이 안 들어감) → 번역을 **슬라이드에 직접 병기**한다. 단, 단일 고정 언어가 아니라 **그 반 학생 명단의 최다 2개 언어**(앱 `get_overview`의 `topLanguages` = 반 학생들이 가장 많이 쓰는 언어 2개)를 **매 시수 생성 프롬프트의 `{병기언어}`로 지정**한다. NotebookLM 디자인 지침엔 *병기 방법(레이아웃)*만 고정으로 박고, *어떤 언어*인지는 프롬프트가 채운다. **(2) 대상은 9~24세 외국인(이주민)으로 통일**. 생성 프롬프트는 참고용 — 과정에 맞게 다듬어 사용.

---

## 0. 대상

- **9~24세 외국인(이주민) 학생**, 한국어를 배우는 중, 컴퓨터를 처음 다룸. 난이도는 **중학생 수준** 기준.
- 컴퓨터·한국어 수준이 반별로 다를 수 있음.
- **쉬운 한국어 + 병기 번역.** 핵심 문구는 한국어를 크게 쓰고, 바로 아래(또는 옆)에 **작은 회색 글씨로 번역을 병기**한다. 병기 언어는 그 반 학생들이 가장 많이 쓰는 **최대 2개 언어**(매 시수 생성 프롬프트의 `{병기언어}`로 지정). 그래도 그림·아이콘·화면 캡처로 이해를 돕는 원칙은 유지한다.

## 1. 디자인 지침 (NotebookLM 소스로 넣기)

```
[슬라이드 디자인 지침 — 공통]
대상은 한국어가 서툰 9~24세 외국 출신(이주민) 학생입니다. 난이도는 중학생 수준. 한눈에 이해되고, 보고 바로 따라 할 수 있는 슬라이드를 만드세요.

전체 분위기
- 밝고 친근한 교육용 스타일. 구조는 단순하게 유지하되 너무 밋밋하지 않게.
- 색상, 아이콘, 카드형 박스, 화면 캡처를 적절히 사용해 지루하지 않게.
- 목표: 쉽다 / 보기 좋다 / 따라 하기 편하다 / 수업이 지루하지 않다.
- 너무 유치하거나 과하게 장식적인 디자인은 피함. 중학생이 보기에 깔끔하고 흥미로운 느낌.

스타일 기준
1. 밝은 배경 + 진한 글자. 한 슬라이드 = 핵심 개념 1개.
2. 제목과 단계 번호(①②③ 또는 1·2·3)는 분명하게 강조.
3. 실습 안내는 카드형 박스로 구성. 버튼 이름은 실제 버튼처럼 [대괄호]로 표시.
4. 중요한 개념은 아이콘과 함께. 순서·동작은 번호와 화살표(→)로 시각화.
5. 글씨는 크고 또렷하게. 멀리서도 읽히게.
6. 화면 캡처와 결과물 예시를 적극 사용(장식용 X, 설명용 O).
7. 여백 넉넉히. 진행 표시(예: 3/10)로 위치 안내. 화면 전환·애니메이션은 최소(어지러운 효과 금지).

색상 기준
- 기본 배경: 흰색 또는 아주 연한 회색.
- 제목 강조: 파란색 또는 보라색 계열(강조는 슬라이드당 한 색).
- 실습 안내 박스: 연한 초록색 또는 하늘색.
- 주의 문구: 연한 주황색 또는 빨간색.
- 완료 기준 박스: 연한 초록색.
- 버튼 이름: 회색 또는 파란색 테두리 박스.
- 피할 것: 너무 강한 원색 조합, 검은 배경 중심, 복잡한 그라데이션, 과한 템플릿, 초등학생용처럼 과하게 귀여운 디자인.

병기 번역 (한국어가 서툰 다국어 학생용)
- 핵심 문구(제목·짧은 안내·버튼 설명·완료 확인 등)는 **한국어를 크게** 쓰고, 바로 아래(또는 옆)에 **작은 회색 글씨로 번역을 병기**한다.
- 병기 언어와 개수는 **생성 프롬프트에서 지정한 언어(최대 2개)** 를 그대로 따른다. 지정이 없으면 병기하지 않고 쉬운 한국어 + 그림으로만 만든다.
- 순서·크기: 한국어(크고 진하게) → 번역(작고 연하게, 보조). 한 슬라이드가 글자로 가득 차지 않게 **핵심 문구에만** 병기하고, 보조 설명은 그림·아이콘으로.
- 번역하지 말 것: 학습 대상이 되는 한글 단어·컴퓨터 용어(예: 따라 입력할 단어, '[저장]' 같은 실제 버튼 이름)는 한글 그대로 둔다.
```

## 2. 형식·언어·구성 (NotebookLM 소스로 넣기)

```
[슬라이드 형식·언어·구성 지침 — 공통]

형식
- 가능하면 Presenter Slides 형식(교사가 말하며 진행, 학생은 핵심만 보고 따라 하기 좋음, 글 과다 방지). 복습/배포용이 필요할 때만 Detailed Deck.

언어
- 한국어 문장은 짧게(약 10~20자), 어려운 단어 금지. 학생에게 직접 말하는 문장.
- 예: "오늘은 사이트를 만듭니다." "먼저 제목을 입력합니다." "저장 버튼을 누릅니다."
- 핵심 문구에는 **지정된 언어(최대 2개)로 번역을 작게 병기**한다(디자인 지침의 '병기 번역' 규칙을 따름). 지정 언어가 없으면 병기하지 않는다.

슬라이드당 정보량
- 한 슬라이드 = 핵심 개념 1개. (좋은 예: '제목 입력하기' / 나쁜 예: 열기+입력+이미지+게시를 한 장에)

조작 설명(단계형)
1) 어디를 볼지 2) 어떤 버튼을 누를지 3) 무엇을 입력할지 4) 결과가 어떻게 보이는지.
- 버튼 이름은 [삽입] [이미지] [게시] [공유] 처럼 [대괄호]로.

실습 슬라이드(각 실습마다 포함)
- 실습 이름 / 완성 목표 / 따라 할 단계(1·2·3) / 완료 확인 기준.
- 예) 실습 이름: 자기소개 제목 만들기 · 완성 목표: 첫 제목 만들기 · 따라하기: 1 제목칸 클릭 2 이름 입력 3 한 문장 쓰기 · 완료 확인: 이름과 소개 문장이 보이면 성공.

결과물 예시 슬라이드
- 최종 실습 전에 '오늘 완성할 결과물'을 반드시 보여줌(완성 모습 미리보기).

수업 종료 안내
- 마무리 인사(종료 안내) 슬라이드는 **만들지 않는다.** 앱에 **상시 표시되는 "수업 마치기" 버튼**이 있어, 교사가 수업 중 아무 때나 눌러 종료 안내를 띄운다.

만들지 말 것
- 글이 너무 많은 슬라이드 / 교사 설명만 있고 학생 행동이 없는 슬라이드 / 추상적 개념만 있는 슬라이드 / 무엇을 눌러야 할지 모르는 슬라이드 / 결과물이 안 보이는 슬라이드 / 화려하지만 흐름이 불명확한 슬라이드.

최종 목표(기준 질문)
- 학생이 지금 무엇을 해야 하는지 바로 아는가?
- 교사가 이 슬라이드만 보고 수업을 진행할 수 있는가?
- 수업 끝에 학생 결과물이 남는가?
```

## 3. NotebookLM 생성 프롬프트 (입력 칸에 붙여넣기 — 참고용, 다듬어 사용)

### 기본 프롬프트
```
업로드된 이번 시수 구성안과 위 디자인·형식 지침을 바탕으로 수업용 Presenter Slides를 만들어 주세요.

대상은 9~24세 외국인(이주민) 학생이며, 난이도는 중학생 수준입니다. 쉬운 한국어로 작성하되, 핵심 문구에는 {병기언어}로 번역을 작게 회색으로 병기해 주세요(최대 2개 언어, 한국어 아래/옆에 보조로). {병기언어}가 비어 있으면 병기하지 말고 쉬운 한국어 + 그림으로만 만들어 주세요. 어려운 말은 그림·아이콘·화면 캡처로 대신 설명해 주세요. ※ {병기언어}는 그 반 학생들이 가장 많이 쓰는 언어로, 강사가 채워 넣습니다(앱 학생 명단의 사용 언어 기준).

스타일은 너무 단순하거나 밋밋하지 않게, 밝고 친근한 교육용 디자인으로 만들되 초등학생용처럼 유치하지 않게 해 주세요. 아이콘, 카드형 박스, 단계 번호, 화면 캡처 활용을 적절히 반영해 주세요.

학생이 화면을 보고 바로 따라 할 수 있게 버튼 이름은 [버튼명]으로 표시하고, 실습 단계는 1·2·3 순서로 명확히 보여 주세요.

각 실습에는 (실습 이름 / 완성 목표 / 따라 할 단계 / 완료 확인 기준)을 포함하고, 최종 실습 전에는 '오늘 완성할 결과물' 예시를 보여 주세요.

수업 종료/마무리 인사 슬라이드는 만들지 마세요. (앱의 '수업 마치기' 버튼으로 대체합니다.)
```

### (선택) 수업 길이 추가
- **60분**: "60분 기준. 슬라이드는 적게, 실습 시간 확보. 핵심 설명은 짧게, 중간 실습 2개 + 최종 실습 1개. 각 실습은 시간 안에 완성할 작은 결과물로."
- **90분**: "90분 기준. 교사 시범·학생 실습·결과 확인 시간을 조금 더 넉넉히. 중간 실습 2개 + 최종 실습 1개 + 스스로 수정·꾸미기 시간. 마지막에 결과 공유/간단 발표."

### 사용 순서
1) 이번 시수 구성안을 소스로 넣기 → 2) 이 문서 [1]·[2]를 소스(또는 커스터마이즈)에 넣기 → 3) Slide Deck 생성, Format = Presenter Slides, Output language = Korean → 4) 입력 칸에 기본 프롬프트(+ 필요 시 길이 추가) 붙여넣기.

## 4. 시수별 16장 이론 프롬프트 (이 과정 자동 파이프라인용) ★ — 정본 템플릿

[1]·[2] 디자인·형식 지침은 NotebookLM에 **고정**. 매 시수 아래 16장 템플릿에 그 시수 값만 갈아끼운다(값은 해당 회차 `details`에서). **이 16장 템플릿이 정본** — 루틴([`cloud-routine-setup.md`](./cloud-routine-setup.md))과 채팅 프로젝트([`chat-project-guide.md`](./chat-project-guide.md)) 모두 이 구조를 그대로 쓴다.

> **운영**: 이 시수별 프롬프트는 **새벽 클라우드 루틴이 실습 HTML과 함께 자동 생성**해 강사 대시보드 '이론 수업'에 **"N번째 이론수업 프롬프트"**(복사 버튼)로 띄운다. 강사는 복사해 NotebookLM에 붙이기만 하면 된다. 수동으로 만들거나 고치려면 채팅 프로젝트에서 `{회차}회차` 명령을 쓴다(폴백). ⚠️ 슬라이드는 **순수 이론(설명)** 까지만 — 학생이 직접 따라 하는 실습 단계·완료 확인 기준은 넣지 않는다(실습은 별도 인터랙티브 HTML, 16번 장에서 연결만).

```
이번 시수의 이론 슬라이드를 16장으로 만들어 주세요. 아래 [각 장 지시]대로 1번부터 16번까지 빠짐없이 만들어 주세요.

[이번 시수 정보]
- 회차/시수: {회차}회차 {시수}시수
- 주제: {시수제목}
- 모듈: {모듈}
- 주요 활동(=오늘 배울 개념들): {주요활동}
- 이어지는 실습에서 만들 결과물: {결과물}
- 병기 번역 언어: {병기언어}   ← 그 반 학생들이 가장 많이 쓰는 언어 최대 2개. 비어 있으면 병기하지 않음.

[공통 규칙]
- 대상: 9~24세 외국인(이주민) 학생, 한국어 배우는 중, 컴퓨터 처음. 쉬운 한국어(한 문장 10~20자).
- **핵심 문구는 한국어를 크게 + {병기언어}로 번역을 작게 회색으로 병기**(최대 2개 언어, 한국어 아래/옆 보조). {병기언어}가 비면 병기하지 말 것. 학습 대상 한글 단어·실제 버튼 이름([대괄호])은 번역하지 말고 한글 그대로.
- 한 장 = 핵심 1개. 그림·아이콘·화면 그림을 충분히. 화면의 버튼·메뉴 이름은 [대괄호]로.
- 이 슬라이드는 '설명'까지만. 학생이 직접 따라 하는 실습 단계·완료 확인 기준은 넣지 말 것(실습은 별도 HTML).
```
