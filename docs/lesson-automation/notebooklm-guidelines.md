# NotebookLM 슬라이드 공통 지침 (이론 슬라이드) — 디자인 · 형식·언어

> **★ 역할 분리(2026-07-03 재구성)**: 이 문서에는 **[1] 디자인 지침(순수 디자인만)** 과 **[2] 형식·언어 지침**만 있다. 슬라이드의 **내용 구성**(19장 배분·장별 내용·개념 끝 배지를 어느 장에 넣는지·회차별 생성 프롬프트)은 이 문서에 없다 — **정본은 [`cloud-routine-setup.md`](./cloud-routine-setup.md)의 이론 19장 설계 규칙**이다(새벽 루틴이 회차별 프롬프트를 자동 생성). 수동으로 만들거나 고칠 때는 채팅 프로젝트의 `{회차}회차` 명령([`chat-project-guide.md`](./chat-project-guide.md))을 쓴다.
>
> **운영**: [1]·[2]를 NotebookLM **소스(또는 커스터마이즈 칸)에 한 번 고정**으로 넣어 두고, 매 회차에는 루틴(또는 채팅 프로젝트)이 만든 **19장 생성 프롬프트**를 입력 칸에 붙인다. 프롬프트가 내용·구성을 다 지시하므로, 이 문서의 고정 소스는 "어떻게 보일지(디자인)"와 "어떤 말투·형식으로 쓸지(형식·언어)"만 책임진다.
>
> **수업 방식(맥락)**: 이론 덱 1개 = 회차 1개(19장 고정, NotebookLM 최대 19장 생성 검증됨). 교사는 덱을 켜 둔 채 1~3장 설명 → 그 개념의 실습(인터랙티브 HTML, [`practice-recipe.md`](./practice-recipe.md)) → 다음 개념 1~3장 → 실습 … 순서로 짧게 끊어 진행한다(인터리브). 통역(교사 음성 실시간 번역)이 함께 돌아간다.
>

## 1. 디자인 지침 (NotebookLM 소스로 넣기 — 고정 ①) ★ 순수 디자인만

> 대상·난이도·수업 방식 같은 배경 설명은 여기 넣지 않는다 — 그건 매 회차 생성 프롬프트(정본: [`cloud-routine-setup.md`](./cloud-routine-setup.md) 5단계)가 전달한다. 이 소스는 "어떻게 보일지"만.

```
[슬라이드 디자인 지침 — 공통]

전체 분위기
- 밝고 친근한 교육용 스타일. 구조는 단순하게 유지하되 너무 밋밋하지 않게.
- 색상, 아이콘, 카드형 박스, 화면 캡처를 적절히 사용해 지루하지 않게.
- 목표: 쉽다 / 보기 좋다 / 알차다 / 수업이 지루하지 않다.
- 너무 유치하거나 과하게 장식적인 디자인은 피함. 깔끔하고 흥미로운 느낌.

레이아웃·타이포그래피
1. 밝은 배경 + 진한 글자.
2. 글씨는 크고 또렷하게. 멀리서도 읽히게.
3. 제목과 단계 번호(①②③ 또는 1·2·3)는 분명하게 강조.
4. 내용은 카드·번호·표로 구조화해 어수선하지 않게. 여백 넉넉히.
5. 중요한 개념은 아이콘과 함께. 순서·동작은 번호와 화살표(→)로 시각화.
6. 진행 표시(예: 3/19)로 위치 안내. 화면 전환·애니메이션은 최소(어지러운 효과 금지).

색상 기준 (색상은 적당히)
- 기본 배경: 흰색 또는 아주 연한 회색.
- 제목 강조: 파란색 또는 보라색 계열(강조는 슬라이드당 한 색).
- 개념 끝 배지·완료 기준 박스: 연한 초록색.
- 주의 문구: 연한 주황색 또는 빨간색.
- 번역(병기) 박스: 연한 보라색(#ede9fe 계열).
- 버튼 이름: 회색 또는 파란색 테두리 박스.
피할 것:
- 너무 강한 원색 조합 / 검은 배경 중심 디자인 / 복잡한 그라데이션
- 장식이 너무 많은 템플릿 / 초등학생용처럼 과하게 귀여운 디자인

병기 번역 박스 (모양)
- 핵심 문구는 한국어를 크게·진하게 쓰고, 바로 아래(또는 옆)에 연한 보라색 박스로 번역을 작게·연하게 병기한다.
- 크기 위계: 한국어(크고 진하게, 주인공) → 번역(작고 연하게, 보조).

개념 끝 배지 (모양)
- 「✅ 개념 끝! → 🛠 실습: {실습 제목}」 배지는 연한 초록색 배경의 큰 배지로, 슬라이드 오른쪽 아래에 매번 같은 모양으로 넣는다(눈에 확 띄게).
- (어느 장에 넣는지는 생성 프롬프트가 지시한다 — 여기서는 모양만.)
```

## 2. 형식·언어 지침 (NotebookLM 소스로 넣기 — 고정 ②)

```
[슬라이드 형식·언어 지침 — 공통]

형식
- Presenter Slides 형식(교사가 말하며 진행). 복습/배포용이 필요할 때만 Detailed Deck.
- Output language = Korean. 총 장수·장별 내용·배지 위치는 매번 입력되는 생성 프롬프트를 그대로 따른다.

언어
- 쉬운 한국어. 한 문장 10~20자, 어려운 단어 금지. 학생에게 직접 말하는 문장.
- 예: "오늘은 파일을 배웁니다." "파일에는 이름이 있습니다." "저장 버튼을 누릅니다."
- 본문 설명 문장은 병기 없이 쉬운 한국어로 쓴다(병기는 아래 '핵심 문구'에만).

병기 번역 (무엇을 병기하나)
- 핵심 문구(제목·개념 이름·핵심 용어·질문·배지)에만 지정된 병기 언어로 번역을 병기한다(디자인 지침의 '병기 번역 박스' 모양).
- 병기 언어와 개수는 생성 프롬프트의 {병기언어}를 그대로 따른다. 지정이 없으면 병기하지 않고 쉬운 한국어 + 그림으로만.
- 보조 설명 문장까지 전부 병기하지 않는다(글이 두 배가 되지 않게).
- 번역하지 말 것: 학습 대상이 되는 한글 단어·컴퓨터 용어(예: 따라 입력할 단어, '[저장]' 같은 실제 버튼 이름)는 한글 그대로 둔다.

표기
- 화면의 버튼·메뉴 이름은 [삽입] [이미지] [게시] [공유] 처럼 [대괄호]로 표시한다.
- 조작 설명은 단계형으로: 1) 어디를 볼지 2) 어떤 버튼을 누를지 3) 무엇을 입력할지 4) 결과가 어떻게 보이는지.
```

## 3. 사용 순서

1. NotebookLM에서 이번 회차 구성안(있으면)을 소스로 넣는다.
2. 이 문서 **[1] 디자인 + [2] 형식·언어**를 소스(또는 커스터마이즈 칸)에 넣는다 — 한 번 넣어 두면 고정.
3. Slide Deck 생성: Format = **Presenter Slides**, Output language = **Korean**.
4. 입력 칸에 **회차별 19장 생성 프롬프트**를 붙여넣는다.
   - 평소: 새벽 루틴이 자동 생성해 강사 대시보드 '이론 수업'에 복사 버튼으로 띄워 준다(정본: [`cloud-routine-setup.md`](./cloud-routine-setup.md)).
   - 수동 폴백: 채팅 프로젝트에서 `{회차}회차` 명령([`chat-project-guide.md`](./chat-project-guide.md)).

> ⚠️ 이 문서의 [1]·[2]를 고치면 NotebookLM에 넣어 둔 고정 소스도 새 버전으로 교체해야 반영된다.
