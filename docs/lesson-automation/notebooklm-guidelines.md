# NotebookLM 슬라이드 공통 지침 (이론 슬라이드)

> **★ 2026-07-03 수업 방식 개편 — 인터리브(짧은 호흡) 진행**: 이론과 실습을 번갈아 진행한다. 교사는 덱을 켜 둔 채 **1~3장 설명 → 그 개념의 실습 열기 → 돌아와서 다음 개념 1~3장 → 실습 …** 순으로 짧게 끊어 간다. 이에 따라:
> ① 이론 덱은 **회차당 1개**(1시수/2시수 분리 폐지 — 쉬는 시간은 교사가 수업 중 판단), ② **19장 고정**(NotebookLM 최대 19장 생성 검증됨 — 16장 아님), ③ **개념 블록 구조**: 개념마다 2~4장, 각 개념의 마지막 장에 **「✅ 개념 끝 → 🛠 실습」 배지**(교사가 실습을 여는 신호), ④ 실습은 **개념당 1개**(이론 1 : 실습 N — 1:1 시수 매칭 아님), ⑤ 통역(교사 음성 실시간 번역)이 함께 돌아가므로 **한 장에 정보를 충분히** 담는다(그림만 고집 X), ⑥ **의미 없이 넘어가는 장 금지**, ⑦ 지난 시간 상기 장은 **이어지는 내용일 때만**(필수 아님).
>
> **대안 경로(2026-06-29):** 이론을 **Claude가 자체 완결 HTML 덱으로 직접 생성**하는 자동화 레시피 → [`theory-html-recipe.md`](./theory-html-recipe.md) (디자인 토큰·19장 구조는 이 문서 §1·§4를 공유). 풍부한 일러스트·실제 스크린샷이 필요하면 아래 NotebookLM 수동 경로가 낫다.

> 이론 슬라이드는 **NotebookLM으로 수동 제작**한다(공개 API 없음 → 자동화 제외).
> 운영: **[1] 디자인 + [2] 형식·언어** 지침을 NotebookLM 소스(또는 커스터마이즈 칸)에 한 번 넣고, 매번 **[4] 회차별 19장 프롬프트**를 입력 칸에 붙인다([3]은 참고용 기본 프롬프트).
> 슬라이드는 교사 발표용이며, 각 개념이 끝날 때마다 **그 개념의 실습(인터랙티브 HTML, [`practice-recipe.md`](./practice-recipe.md))** 으로 연결된다. 대상·톤은 실습과 동일.
>
> **출처/정정**: 사용자의 「NotebookLM 슬라이드 디자인 지침」 + 「슬라이드 생성 프롬프트」 문서를 정리한 것. **(1) 병기 번역 부활(2026-06-25)**: 학생은 슬라이드에 크롬 번역을 쓸 수 없다(슬라이드는 교차출처 임베드라 크롬 페이지 번역이 안 들어감) → 번역을 **슬라이드에 직접 병기**한다. 병기 언어는 **그 반 클래스 설정에 강사가 지정한 병기 언어**(앱 `get_overview`의 `annotationLanguages`, 0개~여러 개. 학생 명단으로 유추하지 않음)를 **매 회차 생성 프롬프트의 `{병기언어}`로 지정**한다. **(2) 대상은 9~24세 외국인(이주민)으로 통일**. **(3) 2026-07-03 인터리브 개편**(위 ★). 생성 프롬프트는 참고용 — 과정에 맞게 다듬어 사용.

## 0. 대상

- **9~24세 외국인(이주민) 학생**, 한국어를 배우는 중, 컴퓨터를 처음 다룸. 난이도는 **중학생 수준** 기준.
- 컴퓨터·한국어 수준이 반별로 다를 수 있음.
- **쉬운 한국어 + 병기 번역.** 핵심 문구는 한국어를 크게 쓰고, 바로 아래(또는 옆)에 **연한 보라색 박스(#ede9fe 계열)에 번역을 병기**한다. 병기 언어는 그 반 **클래스 설정의 병기 언어 전부**(`get_overview`의 반별 `annotationLanguages`, 강사가 직접 지정, 0개~여러 개. 매 회차 생성 프롬프트의 `{병기언어}`로 지정). 학생 language로 유추하지 않는다.
- **통역이 함께 돌아간다(2026-07-03)**: 수업 중 교사 발화가 실시간으로 학생 언어로 번역·전달된다. → 슬라이드를 그림만으로 만들 필요 없다. **한 장에 설명 문장 2~4개 + 그림·도식·표**로 정보를 충분히 담고, 카드·번호·표로 구조화한다.

## 1. 디자인 지침 (NotebookLM 소스로 넣기) — 지침 ①

```
[슬라이드 디자인 지침 — 공통]
대상은 한국어가 서툰 9~24세 외국 출신(이주민) 학생입니다. 난이도는 중학생 수준. 한눈에 이해되는 슬라이드를 만드세요. 수업 중 교사 발화는 실시간 통역으로 번역되어 전달되므로, 슬라이드에 정보를 충분히 담아도 됩니다.

전체 분위기
- 밝고 친근한 교육용 스타일. 구조는 단순하게 유지하되 너무 밋밋하지 않게.
- 색상, 아이콘, 카드형 박스, 화면 캡처를 적절히 사용해 지루하지 않게.
- 목표: 쉽다 / 보기 좋다 / 알차다 / 수업이 지루하지 않다.
- 너무 유치하거나 과하게 장식적인 디자인은 피함. 중학생이 보기에 깔끔하고 흥미로운 느낌.

스타일 기준
1. 밝은 배경 + 진한 글자. 한 슬라이드 = 한 가지 이야기(한 개념의 한 측면).
2. 한 슬라이드에 정보를 충분히: 설명 문장 2~4개 + 그림·도식·표·화면 그림. 단 카드·번호·표로 구조화해 어수선하지 않게.
3. 제목과 단계 번호(①②③ 또는 1·2·3)는 분명하게 강조.
4. 중요한 개념은 아이콘과 함께. 순서·동작은 번호와 화살표(→)로 시각화.
5. 버튼 이름은 실제 버튼처럼 [대괄호]로 표시.
6. 글씨는 크고 또렷하게. 멀리서도 읽히게.
7. 여백 넉넉히. 진행 표시(예: 3/19)로 위치 안내. 화면 전환·애니메이션은 최소(어지러운 효과 금지).

개념 끝 배지 (★ 실습 신호)
- 각 개념의 마지막 슬라이드 오른쪽 아래에 매번 같은 모양의 큰 배지를 넣는다:
  「✅ 개념 끝! → 🛠 실습: {그 개념의 실습 제목}」 (연한 초록색 배경, 눈에 확 띄게)
- 교사가 이 배지를 보고 그 실습을 연다. 개념 중간 장에는 이 배지를 넣지 않는다.

의미 없는 장 금지
- 간지(구호만 있는 장), 앞 내용 단순 반복 장, 채우기용 장을 만들지 않는다.
- 모든 장에 새로운 정보(설명·예시·비교·화면 그림·질문)가 있어야 한다.

색상 기준
색상은 적당히 사용합니다.
- 기본 배경: 흰색 또는 아주 연한 회색.
- 제목 강조: 파란색 또는 보라색 계열(강조는 슬라이드당 한 색).
- 개념 끝 배지·완료 기준 박스: 연한 초록색.
- 주의 문구: 연한 주황색 또는 빨간색.
- 번역(병기) 박스: 연한 보라색.
- 버튼 이름: 회색 또는 파란색 테두리 박스.
피할 것:
- 너무 강한 원색 조합
- 검은 배경 중심 디자인
- 복잡한 그라데이션
- 장식이 너무 많은 템플릿
- 초등학생용처럼 과하게 귀여운 디자인

병기 번역 (한국어가 서툰 다국어 학생용)
- 핵심 문구(제목·개념 이름·핵심 용어·질문·배지 등)는 **한국어를 크게** 쓰고, 바로 아래(또는 옆)에 **연한 보라색 박스에 번역을 병기**한다.
- 병기 언어와 개수는 **생성 프롬프트에서 지정한 언어**를 그대로 따른다. 지정이 없으면 병기하지 않고 쉬운 한국어 + 그림으로만 만든다.
- 순서·크기: 한국어(크고 진하게) → 번역(작고 연하게, 보조). 병기는 핵심 문구에만 — 보조 설명 문장까지 전부 병기해 글이 두 배가 되지 않게 한다(교사 발화는 통역이 실시간 번역해 준다).
- 번역하지 말 것: 학습 대상이 되는 한글 단어·컴퓨터 용어(예: 따라 입력할 단어, '[저장]' 같은 실제 버튼 이름)는 한글 그대로 둔다.
```

## 2. 형식·언어·구성 (NotebookLM 소스로 넣기) — 지침 ②

```
[슬라이드 형식·언어·구성 지침 — 공통]

수업 진행 방식 (이 덱이 쓰이는 방법)
- 이론 덱 1개 = 수업(회차) 1회 전체. 1교시/2교시로 나누지 않는다(쉬는 시간은 교사가 수업 중 판단).
- 교사는 덱을 켜 둔 채 1~3장 설명 → 그 개념의 실습(별도 HTML)을 열어 학생이 해 보고 → 돌아와서 다음 개념 1~3장 → 실습 … 순서로 짧게 끊어 진행한다.
- 그래서 개념마다 마지막 장에 「✅ 개념 끝! → 🛠 실습: {실습 제목}」 배지가 반드시 있어야 한다(디자인 지침 참고).

형식
- 가능하면 Presenter Slides 형식(교사가 말하며 진행). 복습/배포용이 필요할 때만 Detailed Deck.
- 총 19장을 정확히 채운다(최대 19장 생성 가능 확인됨). 의미 없이 넘어가는 장은 만들지 않는다.

언어
- 한국어 문장은 짧게(약 10~20자), 어려운 단어 금지. 학생에게 직접 말하는 문장.
- 예: "오늘은 파일을 배웁니다." "파일에는 이름이 있습니다." "저장 버튼을 누릅니다."
- 핵심 문구에는 **지정된 병기 언어(0개~여러 개)로 번역을 작게 병기**한다(디자인 지침의 '병기 번역' 규칙). 지정 언어가 없으면 병기하지 않는다.
- 교사 발화는 실시간 통역으로 번역되므로, 슬라이드 본문은 병기 없이 쉬운 한국어 설명 문장을 충분히 써도 된다.

슬라이드당 정보량
- 한 슬라이드 = 한 가지 이야기. 단 정보는 충분히: 설명 문장 2~4개 + 그림·도식·표·화면 그림.
- (좋은 예: '파일 이름은 왜 필요할까' 한 장에 이유 2개 + 나쁜 이름/좋은 이름 비교표 / 나쁜 예: "파일!"이라는 제목만 크게 있는 장)

덱 구성 (19장)
1) 표지 1장: 제목 + 회차 + 오늘의 흐름(개념 ①~N과 실습 N개를 지도처럼).
2) 지난 시간 상기 1장: 지난 수업과 이어지는 내용일 때만 만든다. 새 주제면 만들지 않고 그 장수를 개념 설명에 쓴다.
3) 개념 블록(순서대로): 개념마다 2~4장 — 소개·왜 필요한지 → 자세히(화면·동작·예시·비교) → 정리 + 개념 끝 배지. 어렵거나 중요한 개념에 장을 더 배정한다.
4) 마지막 1장: 오늘 전체 정리(개념들을 표·아이콘으로 비교). ⛔ 이해 확인·OX·고르기 등 퀴즈 장은 만들지 않는다(이해 점검은 각 개념의 실습이 담당).

조작 설명(단계형)
1) 어디를 볼지 2) 어떤 버튼을 누를지 3) 무엇을 입력할지 4) 결과가 어떻게 보이는지.
- 버튼 이름은 [삽입] [이미지] [게시] [공유] 처럼 [대괄호]로.

실습과의 관계
- 이 슬라이드는 '설명'까지만. 학생이 직접 따라 하는 실습 단계·완료 확인 기준은 넣지 않는다(실습은 별도 인터랙티브 HTML).
- 개념 끝 배지가 실습으로 넘어가는 유일한 연결 지점이다. 필요하면 그 개념 실습에서 만들 결과물을 '자세히' 장에서 미리 보여줘도 좋다(필수 아님).

수업 종료 안내
- 마무리 인사(종료 안내) 슬라이드는 **만들지 않는다.** 앱에 **상시 표시되는 "수업 마치기" 버튼**이 있어, 교사가 수업 중 아무 때나 눌러 종료 안내를 띄운다.

만들지 말 것
- ⛔ 이해 확인·OX·고르기 등 퀴즈 장(특히 마지막 장) — 이해 점검은 각 개념의 실습이 담당한다.
- 의미 없이 넘어가는 장(간지·구호·단순 반복) / 교사 설명 없이 채우기만 한 장 / 추상적 개념만 있고 예시·화면이 없는 장 / 개념 끝 배지가 없는 개념 블록 / 흐름이 불명확한 장.

최종 목표(기준 질문)
- 교사가 이 슬라이드만 보고 수업을 진행할 수 있는가?
- 교사가 실습을 열 타이밍(개념 끝 배지)이 한눈에 보이는가?
- 19장 모두가 각자 의미 있는 정보를 담고 있는가?
```

## 3. NotebookLM 생성 프롬프트 (입력 칸에 붙여넣기 — 참고용, 다듬어 사용)

### 기본 프롬프트
```
업로드된 이번 회차 구성안과 위 디자인·형식 지침을 바탕으로 수업용 Presenter Slides를 19장으로 만들어 주세요.

대상은 9~24세 외국인(이주민) 학생이며, 난이도는 중학생 수준입니다. 쉬운 한국어로 작성하되, 핵심 문구에는 {병기언어}로 번역을 연한 보라색 박스에 병기해 주세요(반 설정에 지정된 언어 전부, 한국어 아래/옆). {병기언어}가 비어 있으면 병기하지 말고 쉬운 한국어로만 만들어 주세요. ※ {병기언어}는 반 설정(클래스 설정 > 병기 번역 언어, `annotationLanguages`)에 강사가 직접 지정한 언어입니다(학생 사용 언어로 유추하지 않음).

수업은 "이론 1~3장 설명 → 실습 → 다음 개념 1~3장 → 실습" 순서로 진행됩니다. 개념마다 2~4장으로 묶고, 각 개념의 마지막 장 오른쪽 아래에 「✅ 개념 끝! → 🛠 실습: {실습 제목}」 배지를 넣어 주세요.

교사 발화는 실시간 통역으로 번역되므로, 한 장에 설명 문장 2~4개 + 그림·도식·표로 정보를 충분히 담아 주세요. 의미 없이 넘어가는 장은 만들지 마세요.

스타일은 밝고 친근한 교육용 디자인으로, 초등학생용처럼 유치하지 않게 해 주세요. 아이콘, 카드형 박스, 단계 번호, 화면 캡처를 적절히 활용해 주세요. 버튼 이름은 [버튼명]으로 표시해 주세요.

수업 종료/마무리 인사 슬라이드는 만들지 마세요. (앱의 '수업 마치기' 버튼으로 대체합니다.) 이해 확인·OX·고르기 같은 퀴즈 장도 만들지 마세요(이해 점검은 실습이 담당합니다).
```

### 사용 순서
1) 이번 회차 구성안을 소스로 넣기 → 2) 이 문서 [1]·[2]를 소스(또는 커스터마이즈)에 넣기 → 3) Slide Deck 생성, Format = Presenter Slides, Output language = Korean → 4) 입력 칸에 [4]의 회차별 19장 프롬프트(권장) 또는 위 기본 프롬프트 붙여넣기.

## 4. 회차별 19장 이론 프롬프트 (이 과정 자동 파이프라인용) ★ — 정본 템플릿

[1]·[2] 디자인·형식 지침은 NotebookLM에 **고정**. 매 회차 아래 19장 템플릿에 그 회차 값만 갈아끼운다. **이 19장 템플릿이 정본** — 루틴([`cloud-routine-setup.md`](./cloud-routine-setup.md))과 채팅 프로젝트([`chat-project-guide.md`](./chat-project-guide.md)) 모두 이 구조를 그대로 쓴다.

**값 채우는 법(개념 도출)**: 그 회차 `details`의 시수 블록들을 **순서대로 합쳐** '주요 활동' 항목들을 **개념 목록**으로 만든다(1시수 활동들 → 2시수 활동들 순). 적정 **3~6개** — 7개 이상이면 연관 항목을 묶고, 2개 이하면 나눠서 3개 이상으로. 개념마다 짧은 제목을 붙인다. **개념마다 실습 HTML 1개가 먼저 만들어져 있어야** 하고(순서 동일, [`practice-recipe.md`](./practice-recipe.md)), 그 실습 제목을 배지에 넣는다.

> **운영**: 이 회차별 프롬프트는 **새벽 클라우드 루틴이 실습 HTML(개념당 1개)과 함께 자동 생성**해 강사 대시보드 '이론 수업'에 프롬프트(복사 버튼)로 띄운다. 강사는 복사해 NotebookLM에 붙이기만 하면 된다. 수동으로 만들거나 고치려면 채팅 프로젝트에서 `{회차}회차` 명령을 쓴다(폴백). ⚠️ 슬라이드는 **순수 이론(설명)** 까지만 — 학생이 직접 따라 하는 실습 단계·완료 확인 기준은 넣지 않는다(실습은 별도 인터랙티브 HTML, 개념 끝 배지로 연결만).
