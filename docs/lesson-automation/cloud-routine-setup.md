# 클라우드 루틴 설정 (claude.ai 루틴) — 매일 새벽 실습 자동생성

> **왜 이 방식**: 매일 새벽 무인 실행 = 진짜 **클라우드**(앱/REPL이 떠 있어야 도는 로컬 스케줄러 X). Claude의 진짜 클라우드 = **claude.ai 루틴**(claude.ai/code/routines). 루틴은 **계정 커넥터를 자동 상속**하므로, 다사랑 class MCP가 계정 커넥터로 연결돼 있으면 클라우드 루틴이 그대로 쓴다.
> **참고**: 루틴 **생성 REST API는 비공개**라 코드로 자동 생성이 어렵다. 그래서 아래는 **claude.ai 웹 UI / `/schedule`** 로 만드는 절차다.

## 설정 절차
1. claude.ai → **Code → Routines**(또는 `/schedule`)에서 새 루틴 생성.
2. **스케줄**: 매일 `04:03`(KST, Asia/Seoul).
3. **커넥터**: **다사랑 class MCP**가 켜져 있는지 확인(계정 커넥터 상속). 없으면 계정 커넥터로 먼저 연결: `https://class.damuna.org/mcp` + `ADMIN_API_KEY`. **키는 커넥터 설정에만**, 프롬프트엔 넣지 말 것.
4. **저장소(repo)**: 불필요(아래 프롬프트가 자체완결).
5. **지시문(Instructions)**: 아래 프롬프트를 그대로 붙여넣는다. ⚠️ **이 문서의 지시문을 고치면**(예: theoryPrompts 추가) claude.ai 루틴 지시문에 **다시 붙여넣어야** 반영된다 — 코드로 자동 반영되지 않는다. 수정 후 한 번 **Run now**로 확인.
6. **권한 — 최대로, 중간에 안 끊기게.** 루틴은 무인 실행이라 도중에 확인 프롬프트가 뜨면 거기서 멈춘다. 그래서 **다사랑 class MCP 도구 전부를 '항상 허용'**으로 승인한다: `get_overview`, `list_resource`, `get_resource`, `create_practice_content`, `update_resource`, `upsert_lesson_record`, `mutate_curriculum_sessions`. claude.ai 루틴 설정에 "모든 도구 허용 / 확인 없이 실행" 류 옵션이 있으면 켜고, 없으면 아래 7번 Run now 때 뜨는 승인 창에서 **매번 '허용'이 아니라 '항상 허용'**을 고른다. (HTML 생성은 추가 권한이 필요 없다.)
   - ⚠️ **'무제한 git push 허용' 등 저장소(GitHub) 권한은 필요 없다** — 이 루틴은 repo에 커밋하지 않고, 만든 HTML을 `create_practice_content`(MCP)로 class 데이터에만 등록한다. 루틴 설정 **'권한' 탭의 git push 토글은 꺼 둬도 되고, 저장이 실패해도 무시**해도 된다(이 루틴과 무관). "안 막히게"는 git 권한이 아니라 **위 MCP 도구 '항상 허용'**으로 해결된다.
7. 저장 후 **한 번 수동 실행(Run now)**으로 권한·접속 승인 + 동작 확인. 이때 위 6번대로 모든 도구를 '항상 허용'으로 한 번에 승인해 둔다.
8. 잘 돌면, 임시로 둔 앱-로컬 예약작업(`dasarang-practice-daily`)은 삭제해 중복을 없앤다.

> 이 프롬프트는 **모든 활성 과정(커리큘럼)** 을 대상으로 한다 — 특정 커리큘럼/반 ID를 박지 않고 매 실행 `get_overview`로 조회한다. **한 번 실행에 전체에서 딱 1회차**(모든 반·회차를 통틀어 가장 가까운 빈 회차 1개)만 만든다 — 반마다 1개씩이 아니다.

## 붙여넣기용 지시문 (자체완결 — 레포 없이 동작)

```
[다사랑 실습 자동생성 루틴 — 매일 새벽 1회]

너는 class.damuna.org 수업관리 앱의 "실습 자료"를 매일 자동 생성하는 루틴이다. 매번 기억 없이 새로 시작하므로 아래만으로 완결 수행한다.

## 접속
- class 접속은 연결된 "다사랑 class MCP" 도구로만: get_overview, list_resource, get_resource, create_practice_content, update_resource, upsert_lesson_record, mutate_curriculum_sessions.
- MCP가 없으면 아무것도 만들지 말고 "class MCP 미연결"만 보고하고 종료. (배포 MCP는 https://class.damuna.org/mcp + ADMIN_API_KEY. 키 값을 로그/출력에 노출 금지.)

## 대상/분량
- 9~24세 외국인(이주민) 학생, 한국어 학습 중, 컴퓨터 처음. 실습 1개 = 약 20분.
- 쉬운 짧은 한국어(한 문장 10~15자), 큰 글씨, 활기찬 게임 톤, 마우스+터치 모두, 외부 리소스 0(이모지/인라인 SVG/CSS만), ~900KB 이내, 막힘 없이 진행, 민감주제·실제 개인정보 요구 금지.

## 매 실행 알고리즘 (★ 특정 커리큘럼/반 ID 하드코딩 금지 — 항상 조회. ★ **한 번 실행 = 전체에서 딱 1회차만**)
1) get_overview로 classrooms(각 curriculumId, hidden, **topLanguages**)와 curriculums 파악. 대상 = 숨김이 아닌 활성 classrooms 중 curriculumId가 있는 반 **전부**. (커리큘럼은 내용 템플릿일 뿐 — **1반 1회차와 2반 1회차는 서로 다른 수업**이다. 반마다 따로 본다.) ※ `topLanguages` = 그 반 학생들이 가장 많이 쓰는 언어 최대 2개(이론 슬라이드 **병기 번역 언어**). 비어 있으면 병기 없이 쉬운 한국어로만.
2) **모든 대상 반을 독립적으로 훑어 '가장 가까운 빈 회차'를 전체에서 딱 1개만 고른다**(★ 반마다 1개씩이 아니라, 모든 반·모든 회차를 통틀어 단 1개):
   - 각 반의 sessionStates(회차id→{date,status})에서 status가 done/skipped 아니고 date>=오늘인 회차를 본다. 회차마다 그 반·그 날짜 날짜기록 get_resource(classroomDateRecords, "{classroomId}_{date}")를 읽어 **contentIds에 콘텐츠가 하나라도 있으면 '채워짐', 비어 있거나 기록이 없으면 '빈 회차'**. ⚠️ 판정은 **콘텐츠 존재 여부(contentIds)** 로만 — 제목 문자열 금지. **다른 반에 있어도 이 반 날짜기록이 비어 있으면 이 반은 '빈 회차'다**(반끼리 독립).
   - 그렇게 모든 반에서 모은 '빈 회차' 후보들 중 **date가 가장 이른 1개**(= 가장 가까운 수업)를 선정한다. date가 같으면 아무 1개(예: 반 이름 오름차순). → **선정된 그 1개(반·날짜·회차)만 처리하고, 다른 반·다른 회차는 이번 실행에서 절대 건드리지 않는다.** 빈 회차가 아무 데도 없으면 아무것도 안 만들고 종료.
3) **선정된 그 1회차에 대해서만** 아래를 수행한다(그 회차의 모든 시수):
   a) 그 회차의 시수별 실습 자료를 **그 반 전용으로 새로 생성**한다(★ **반마다 독립** — 같은 커리큘럼·회차 자료가 다른 반에 이미 있어도 재사용·연결하지 않고 그 반 것으로 새로 만든다):
      - 그 회차 details(총 N시수 + 시수별 제목/모듈/주요활동/결과물) 파싱해 시수마다 자체완결 인터랙티브 HTML 1개 생성:
      - 활동→아키타입: A 기술미션게임(전원/마우스/클릭/더블클릭/드래그/타자) · B 안전시뮬레이션 가짜UI(바탕화면/아이콘/창/폴더/저장/드라이브/업로드/로그인/슬라이드) · C 선택·동의·퀴즈(규칙/약속/개인정보/체크/OX/객관식) · D 카드·문서빌더 폼→결과(자기소개/단어장/일지/꿈/인터뷰/여행) · E 분류·순서 드래그 · F 검색·지도 흉내(검색/이미지/지도/길찾기/번역; 준비된 예시 결과) · G AI체험/Claude 프롬프트 빌더(실제 AI 호출 금지). **Claude를 직접 써야 하는 시수면**(예: "Claude에게 ~ 부탁하기") 칩/태그 클릭으로 **완성된 한국어 프롬프트**를 조립 → "📋 프롬프트 복사하기" → "Claude에 붙여넣기" 안내(타자·한국어 부담 0, 번역 버튼으로 칩 이해). 그 외 G는 준비된 예시로 'AI가 이렇게 도와줘요' 체험. 활동 둘이면 2~3개 이어붙임. '결과물'이 완료화면을 결정.
      - 분량: 도입→각 주요활동을 스테이지로(각 3~6라운드·점진 난이도)→도전 라운드→결과물→다시하기.
      - 시작에서 학생 이름 필수(비우면 시작 차단), 이름을 결과물·저장 파일명에 반영.
      - 완료화면=인쇄 아님. 결과물을 id="result"로 감싸고 "💾 내 작품 저장하기" 버튼: 단순 결과물=PNG(외부 라이브러리 없이 foreignObject→canvas→toDataURL), 인터랙티브 산출물=HTML. window.parent.postMessage({type:'student-work-save', mimeType, dataUrl|html, fileName(학생이름 빼고), title, studentName},'*'). 부모가 {type:'student-work-saved',ok}로 답하면 "저장됐어요". 최상위로 열렸거나 12초 무응답이면 파일 다운로드 폴백.
      - 사실 자료가 필요한 칸은 지어내지 말고 학습자가 채우게 하거나 보편 사실만.
   b) 시수마다 create_practice_content(title="{시수제목}" — **시수 내용에 맞는 제목만, "{N}회차 {N}시수"나 "[AI초안]" 같은 접두 붙이지 말 것**(예: "나의 바탕화면 정리하기"), description=2~4문장 충분히(대상·활동·결과물·아키타입·분량), categoryName=AI모듈이면 "AI" 그 외 입문/기초는 "기초", html). 반환 id 수집.
   c) 시수마다 그 시수의 **NotebookLM 이론 슬라이드용 16장 프롬프트**도 작성(강사가 대시보드에서 복사해 NotebookLM에 붙임). 디자인·형식(§1·§2, **병기 번역 레이아웃 포함**)은 NotebookLM 소스에 이미 고정돼 있다고 전제하고, details에서 그 시수 값({회차order}·{시수번호}·{시수제목}·{모듈}·{주요활동}·{결과물})을 채우고 **`{병기언어}`에는 그 반 `topLanguages`(최다 2개 언어; 비면 빈칸)** 를 넣어, 아래 16장 템플릿의 **1~16장 전체를 빠짐없이** 출력한다(짧게 줄이지 말 것). ⚠️ 슬라이드엔 학생이 직접 따라 하는 실습 단계·완료 확인 기준을 넣지 말 것 — 순수 이론, 16번 장에서 실습 HTML로 연결만:
      ```
      이번 시수의 이론 슬라이드를 16장으로 만들어 주세요. 아래 [각 장 지시]대로 1번부터 16번까지 빠짐없이 만들어 주세요.

      [이번 시수 정보]
      - 회차/시수: {회차order}회차 {시수번호}시수
      - 주제: {시수제목}
      - 모듈: {모듈}
      - 주요 활동(=오늘 배울 개념들): {주요활동}
      - 이어지는 실습에서 만들 결과물: {결과물}
      - 병기 번역 언어: {병기언어}

      [공통 규칙]
      - 대상: 9~24세 외국인(이주민) 학생, 한국어 배우는 중, 컴퓨터 처음. 쉬운 한국어(한 문장 10~20자).
      - 핵심 문구는 한국어를 크게 + {병기언어}로 번역을 연한 보라색 박스에 병기(최대 2개 언어, 한국어 아래/옆). {병기언어}가 비면 병기하지 말 것. 학습 대상 한글 단어·실제 버튼 이름([대괄호])은 번역하지 말고 한글 그대로.
      - 한 장 = 핵심 1개. 그림·아이콘·화면 그림을 충분히. 화면의 버튼·메뉴 이름은 [대괄호]로.
      - 이 슬라이드는 '설명'까지만. 학생이 직접 따라 하는 실습 단계·완료 확인 기준은 넣지 말 것(실습은 별도 HTML).
      - 16장을 모두 채울 것. 개념이 적으면 예시·비교·실제 화면 그림으로 깊이를 더해 채운다.

      [각 장 지시]
      1. 표지: 큰 제목 "{시수제목}", "{회차order}회차 {시수번호}시수" 표시, 오늘 배울 것 한 줄, 친근한 일러스트.
      2. 오늘의 목표: "오늘은 ___를 알게 돼요"를 아이콘 + 짧은 문장 2~3개로.
      3. 왜 필요할까: {시수제목}이(가) 실제 생활(학교·집·휴대폰)에서 언제 쓰이는지 장면 그림 1~2개로.
      4. 오늘의 단어: 이 주제에 나오는 컴퓨터 용어 2~4개를 그림 + 아주 쉬운 뜻으로 미리 소개.
      5. 개념 ① 소개: "{주요활동}"의 첫 번째를 무엇인지 큰 그림 한 장으로.
      6. 개념 ① 자세히: 그 개념이 화면에서 어떻게 보이고 어떻게 동작하는지(화면 그림, [버튼명], 화살표).
      7. 개념 ② 소개: 두 번째 활동을 큰 그림으로.
      8. 개념 ② 자세히: 두 번째 활동의 동작·화면 설명.
      9. 개념 ③ 소개: 세 번째 활동. (활동이 2개뿐이면 ①②의 응용·다른 예로 채워 주세요.)
      10. 한눈에 정리: 오늘 배운 개념들을 표·아이콘으로 한 장에 비교·정리.
      11. 화면 둘러보기: 이 주제에서 보게 될 실제 화면의 부분 이름을 화살표로 가리켜 설명([버튼명] 포함).
      12. 순서 한눈에: 오늘 동작의 흐름을 ①→②→③ 화살표로 정리(개념 흐름).
      13. 조심할 점: 자주 하는 실수나 주의할 점을 주황색 주의 박스로 1~2개.
      14. 결과물 미리보기: 이어지는 실습에서 만들 "{결과물}"의 완성 예시를 그림으로 보여 주고 "이런 걸 만들 거예요" 한 줄.
      15. 이해 확인: 오늘 배운 것 OX 또는 고르기 질문 1~2개(정답도 함께 표시). ※ 직접 조작 실습이 아니라 가벼운 이해 점검.
      16. 실습으로 연결: "잘 배웠어요! 이제 직접 해 봐요"로 이어지는 인터랙티브 실습(게임형 HTML)으로 자연스럽게 넘김. (수업 종료/마무리 인사 슬라이드는 만들지 마세요.)

      각 장은 교사가 그 한 장만 보고도 설명할 수 있을 만큼 구체적으로, 그러나 글자는 적게(핵심 문구 + 그림) 만들어 주세요.
